## تعارف

AI معاون ایک Python پر مبنی مقامی سرور ایپلیکیشن ہے جو آپ کو روزمرہ کے کاموں کو خودکار بنانے، فائلوں کو منظم کرنے، سسٹم کمانڈز چلانے اور N8N ورک فلوز کے ساتھ مربوط کرنے میں مدد کرتی ہے۔ اس میں شامل ہے:

- **قدرتی زبان کی پروسیسنگ**: انگریزی اور اردو میں احکام سمجھتا ہے
- **کام کا انتظام**: یاد دہانیوں کے ساتھ کام بنائیں، ٹریک کریں اور مکمل کریں
- **فائل آپریشنز**: ورک اسپیس کے اندر محفوظ فائل ہیرا پھیری
- **سسٹم کمانڈز**: محفوظ طریقے سے وائٹ لسٹڈ سسٹم کمانڈز چلائیں
- **N8N انضمام**: ورک فلوز ٹرگر کریں اور پیچیدہ عمل کو خودکار بنائیں
- **ویب انٹرفیس**: تعامل کے لیے سادہ ویب UI
- **REST API**: پروگرام سے رسائی کے لیے مکمل API

## تنصیب

### مرحلہ 1: پیشگی تقاضے

یقینی بنائیں کہ آپ کے پاس ہے:
- Python 3.10 یا اس سے اوپر
- pip (Python پیکیج مینیجر)
- (اختیاری) ورک فلو آٹومیشن کے لیے N8N

### مرحلہ 2: معاون کو انسٹال کریں

```bash
# ai-assistant ڈائریکٹری میں جائیں
cd ai-assistant

# ورچوئل ماحول بنائیں
python -m venv venv

# ورچوئل ماحول کو چالو کریں
# Linux/Mac پر:
source venv/bin/activate
# Windows پر:
venv\Scripts\activate

# انحصار انسٹال کریں
pip install -r requirements.txt
```

### مرحلہ 3: ماحول کو ترتیب دیں

```bash
# مثال کے ماحولیاتی فائل کو کاپی کریں
cp .env.example .env

# اپنی ترتیبات کے ساتھ .env میں ترمیم کریں
nano .env  # یا اپنے پسندیدہ ایڈیٹر کا استعمال کریں
```

ترتیب دینے کی اہم ترتیبات:
- `WORKSPACE_DIR`: فائل آپریشنز کے لیے ڈائریکٹری
- `N8N_WEBHOOK_URL`: آپ کا N8N webhook بنیادی URL
- `N8N_API_KEY`: آپ کی N8N API کلید (اگر API استعمال کر رہے ہیں)
- `DEFAULT_LANGUAGE`: ڈیفالٹ زبان (en یا ur)

### مرحلہ 4: معاون چلائیں

```bash
python main.py
```

معاون `http://localhost:8001` پر شروع ہو جائے گا

## ترتیب

### ماحولیاتی متغیرات

حسب ضرورت بنانے کے لیے `.env` فائل میں ترمیم کریں:

#### ایپلیکیشن کی ترتیبات
- `APP_NAME`: ایپلیکیشن کا نام
- `PORT`: سرور پورٹ (ڈیفالٹ: 8001)
- `DEBUG`: ڈیبگ موڈ (True/False)

#### سیکیورٹی
- `SECRET_KEY`: سیکیورٹی کے لیے خفیہ کلید (پروڈکشن میں تبدیل کریں)
- `ALLOWED_ORIGINS`: CORS اجازت یافتہ ماخذ

#### ڈیٹا بیس
- `DATABASE_PATH`: SQLite ڈیٹا بیس کا راستہ

#### N8N انضمام
- `N8N_WEBHOOK_URL`: N8N webhook URL
- `N8N_API_URL`: N8N API URL
- `N8N_API_KEY`: آپ کی N8N API کلید

#### زبان
- `DEFAULT_LANGUAGE`: ڈیفالٹ زبان (en یا ur)
- `SUPPORTED_LANGUAGES`: سپورٹ شدہ زبانوں کی کوما سے الگ کردہ فہرست

#### فائل آپریشنز
- `WORKSPACE_DIR`: محفوظ ورک اسپیس ڈائریکٹری
- `MAX_FILE_SIZE`: زیادہ سے زیادہ فائل سائز بائٹس میں
- `ALLOWED_FILE_EXTENSIONS`: اجازت یافتہ ایکسٹینشنز کی کوما سے الگ کردہ فہرست

#### سسٹم کمانڈز
- `ENABLE_SYSTEM_COMMANDS`: کمانڈ عمل کو فعال/غیر فعال کریں
- `SAFE_COMMANDS`: کوما سے الگ کردہ محفوظ کمانڈز
- `BLOCKED_COMMANDS`: کوما سے الگ کردہ بلاک شدہ پیٹرن

## استعمال کی رہنمائی

### ویب انٹرفیس استعمال کرنا

1. اپنے براؤزر میں `http://localhost:8001` کھولیں
2. اپنی زبان منتخب کریں (انگریزی یا اردو)
3. ان پٹ باکس میں کمانڈز ٹائپ کریں
4. Enter دبائیں یا Send پر کلک کریں

### API استعمال کرنا

#### کام بنائیں

```bash
curl -X POST http://localhost:8001/api/v1/assistant \
  -H "Content-Type: application/json" \
  -d '{
    "message": "کل گروسری خریدنے کا کام بنائیں",
    "language": "ur"
  }'
```

#### کاموں کی فہرست

```bash
curl http://localhost:8001/api/v1/tasks
```

#### فائل آپریشنز

```bash
# فائل بنائیں
curl -X POST http://localhost:8001/api/v1/files/create \
  -H "Content-Type: application/json" \
  -d '{
    "operation_type": "create",
    "file_path": "notes.txt",
    "content": "میرے نوٹس یہاں",
    "language": "ur"
  }'

# فائل پڑھیں
curl -X POST http://localhost:8001/api/v1/files/read \
  -H "Content-Type: application/json" \
  -d '{
    "operation_type": "read",
    "file_path": "notes.txt"
  }'
```

### قدرتی زبان کی کمانڈز

معاون مختلف قدرتی زبان کے پیٹرن سمجھتا ہے:

#### کام کی کمانڈز
- "کام بنائیں [تفصیل]"
- "ٹاسک بنائیں: [تفصیل]"
- "مجھے یاد دلائیں [تفصیل]"
- "میرے کام دکھائیں"
- "زیر التوا کام دکھائیں"
- "کام مکمل کریں [id یا نام]"

#### فائل کمانڈز
- "[فائل نام] نام کی فائل بنائیں"
- "[فائل نام] فائل پڑھیں"
- "[فائل نام] فائل ایڈٹ کریں"
- "مجھے [فائل نام] فائل دکھائیں"

#### سسٹم کمانڈز
- "[کمانڈ] کمانڈ چلائیں"
- "[کمانڈ] رن کریں"

#### N8N کمانڈز
- "[ورک فلو نام] ورک فلو چلائیں"
- "[ورک فلو نام] n8n چلائیں"

## N8N انضمام

### N8N سیٹ اپ کرنا

1. N8N انسٹال کریں:
```bash
npm install -g n8n
```

2. N8N شروع کریں:
```bash
n8n start
```

3. `http://localhost:5678` پر N8N تک رسائی حاصل کریں

### ورک فلوز درآمد کرنا

1. `n8n_workflows` ڈائریکٹری میں جائیں
2. N8N ویب انٹرفیس کھولیں
3. **Import from File** پر کلک کریں
4. ورک فلو JSON فائل منتخب کریں
5. ورک فلو کو چالو کریں

### مثال کے ورک فلوز

#### کام کی آٹومیشن
جب کام بنائے جاتے ہیں تو ٹرگر ہوتا ہے اور فوری کاموں کے لیے اطلاعات بھیجتا ہے۔

Webhook: `http://localhost:5678/webhook/task-created`

#### ای میل آٹومیشن
قدرتی زبان کی کمانڈز کی بنیاد پر ای میل بھیجتا ہے۔

Webhook: `http://localhost:5678/webhook/send-email`

## مسائل کا حل

### معاون شروع نہیں ہو رہا

**مسئلہ**: سرور شروع ہونے میں ناکام

**حل**:
1. چیک کریں کہ پورٹ 8001 دستیاب ہے
2. تصدیق کریں کہ ورچوئل ماحول چالو ہے
3. یقینی بنائیں کہ تمام انحصار انسٹال ہیں: `pip install -r requirements.txt`
4. کنسول آؤٹ پٹ میں غلطیوں کی جانچ کریں

### کام نہیں بن رہے

**مسئلہ**: کام بنانے کی کمانڈز کام نہیں کرتیں

**حل**:
1. ڈیٹا بیس کی موجودگی چیک کریں: .env میں `DATABASE_PATH` کی تصدیق کریں
2. ڈیٹا بیس کی اجازت کی جانچ کریں
3. معاون کو دوبارہ شروع کرنے کی کوشش کریں
4. غلطیوں کے لیے API لاگز چیک کریں

### فائل آپریشنز ناکام

**مسئلہ**: فائلیں نہیں بنا یا پڑھ سکتے

**حل**:
1. تصدیق کریں کہ `WORKSPACE_DIR` موجود ہے اور قابل تحریر ہے
2. فائل کی اجازت چیک کریں
3. یقینی بنائیں کہ فائل کے راستے میں `..` نہیں ہے
4. تصدیق کریں کہ فائل ایکسٹینشن `ALLOWED_FILE_EXTENSIONS` میں ہے

### N8N انضمام کام نہیں کر رہا

**مسئلہ**: Webhooks ٹرگر نہیں ہو رہے

**حل**:
1. تصدیق کریں کہ N8N چل رہا ہے: `http://localhost:5678`
2. .env میں `N8N_WEBHOOK_URL` چیک کریں
3. یقینی بنائیں کہ ورک فلو N8N میں چالو ہے
4. curl کے ساتھ webhook کو براہ راست ٹیسٹ کریں

### سسٹم کمانڈز بلاک ہیں

**مسئلہ**: کمانڈز چلتی نہیں ہیں

**حل**:
1. چیک کریں کہ .env میں `ENABLE_SYSTEM_COMMANDS=True` ہے
2. تصدیق کریں کہ کمانڈ `SAFE_COMMANDS` کی فہرست میں ہے
3. یقینی بنائیں کہ کمانڈ `BLOCKED_COMMANDS` پیٹرن سے میل نہیں کھاتی
4. محفوظ فہرست میں کمانڈ شامل کرنے کی کوشش کریں

## بہترین طریقے

1. **سیکیورٹی**:
   - `SECRET_KEY` کو محفوظ اور منفرد رکھیں
   - صرف ضروری سسٹم کمانڈز کو وائٹ لسٹ کریں
   - باقاعدگی سے `ALLOWED_FILE_EXTENSIONS` کا جائزہ لیں

2. **کارکردگی**:
   - ورک اسپیس ڈائریکٹری کو منظم رکھیں
   - مکمل شدہ کاموں کو باقاعدگی سے صاف کریں
   - ڈیٹا بیس سائز کی نگرانی کریں

3. **N8N انضمام**:
   - چالو کرنے سے پہلے ورک فلوز کی مکمل جانچ کریں
   - ورک فلوز میں خرابی کی ہینڈلنگ استعمال کریں

4. **بیک اپ**:
   - SQLite ڈیٹا بیس کا باقاعدگی سے بیک اپ لیں
   - اہم فائلوں کی کاپیاں رکھیں
   - N8N ورک فلوز کو وقتاً فوقتاً برآمد کریں
